当然这个页面你也可以把它分成一个配置，这个配置就会有了一个新的模板的嵌套，从而你可以搭建更多网页的元素，比方说新建于一个自己的网页版的简历，充一些自己的内容。那么就是这样一个简单的理念，我们可以用原子设计理念来进行一个规范，它就是一个设计系统的基础指导理论，还可以帮助我们的设计师构建产品和用户互动的关系。并且它定义了五层结构，原子到分子到组织，再到模板再到页面。而页面可以在我们整个写作系统里面作为最终的文章产出。我们写文章也可以有套路，有对应的模板去加速我们写作输出的流程。

当然在文章成型之前，我们大多数时候是在积累阅读笔记卡片和自己的闪练想法灵感等等。而这三类在我之前开篇的时候就提到我们到底该以一个什么维度、怎么样的颗粒度来组织我们的卡片，所以这就是我们本期要回答的一个问题，这也是我在 ROI search 里面结合具体的用例来给大家一个我自己的答案，欢迎大家来探讨。

那么介绍完了这个原子化设计之后，我还是想更进一步来给大家讲为什么在软件开发和设计的角度，原子化是非常重要的。其实最主要的原因就是这个正交性。正教性其实就是当每个组件遵循单一职责的原则之后，我们就可以更好地去复用它，也就是组件越小越好，越有能组合的一个可能性。假设一个组件能够做 M 键独特的事情，而另外一个组件能做 N 键不一样的事情。那如果它们是正交的，组合起来就能够做 M 乘以 N 键式。但如果它们不正交有重叠的地方，那自然他们组合起来就有更少的复用程度。所以说单正交性越好，那单个单元也就是单个原子或者卡片或者叫元素，它的性价比就提高了。所以对比 ROI research 的 block 我接下来会给大家讲在 ROI search 里面正调性是怎么体现的。

当我们在使用卡片或者叫描述卡片的时候，我觉得卡片的颗粒度其实是分子层面的，而分子就是在 ROI search 里面的一个个 block 那么分子是由原子构成的，于是分子它是可以由原子符号进行一个自由组合。比方说最常见的这个双链符号就是两个中括号这样的方式，它就是一个原子。而对应的还有圆括号和花括号。当然除了 ROI search 里面这些特殊符号以外，还会有对应的 bug done 的特殊符号，比方说标题或者图片或者链接，而这些符号其实又是可以互相组合的。

比方说当它组合成一个个分子的时候，我们可以用中括号圆括号的方式来表达它的引用关系，也可以用花括号加中括号再加 todo 的方式来表示。这是一个任务，这个任务是可以被完成的，可交互的以及可以通过圆括号中括号番茄的方式来创建一个目标倒计时。于是你就可以看到这边所对应的这个 todo 它就可以进行点击，而这个番茄中它就可以进行一个倒计时。

那么最核心的部分是我想要在今天分享的是这个组织，也就是 organize 有机体这样一个概念，从细胞或者叫人体的角度来解释它当我们有了一个个原子组成分子或者叫细胞之后，一个组织它其实是人的大脑或者心脏或者手这样的一个个组织，也就是它有一定的功能性，而这个功能性恰恰是发挥笔记软件最具特点的地方。

那我们再看第六点，卡片作为分子。前面提到 block 其实是笔记层面的最小单元。当我们把笔记写得相对较小，并且遵循单一职责原则，那么这样合适的一个颗粒度，它就产生了正交性的一个价值。正交性可以让这些组件能够相互组合。那么它组合起来之后，单个卡片的性价比也就提高了。

另外我们可以类比一下大脑的神经元，我们的大脑里面，它并不是一个神经元保存一个单位记忆。相反这些记忆其实是通过神经元和整个网络进行传递的，也就是一个想法，它可以被整个大脑共用。于是乎，当我们在记笔记的时候，当一个笔记工具成为你的第二大脑的时候，单个卡片也应该要被整个第二大脑所共用。而以往像 notion 或者是音像笔记，它都是通过文件夹也就是树状结构来进行信息的组织的。于是每次对单个笔记进行修改的时候，都必须回到它所在的位置进行跟踪和更新，这其实就是非常的冗余。首先你寻找起来非常的难，并且很有可能你重复地去记录同样的想法。另外当一个笔记进行变更之后，其他地方并不能看到对应的变更。还有一个点就是当你移动某个笔记之后，它就缺失失了这个笔记在原本的上下文的解释了。所以你需要双向链接，并且是在足够细的卡片维度进行双向链接。

如果说单个 block 节点可以在不同的位置，也就是同时在不同的序列以及层次结构或页面路径中保持路径的位置，也就是这里的面包屑，那么它就可以跟其他的笔记进行对话，从而形成整个网络。这个网络就是一个动态的，任何笔记的更新或者修改都可以实时地体现在整个网络上，从而更加有可能产生远程的灵感，产生远距联想。
